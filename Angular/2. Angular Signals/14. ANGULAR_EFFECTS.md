# Effects in Angular (Signals)

## Introduction

Effects are a core part of Angular's Signals system.

They are used to **react to signal changes inside TypeScript code** and execute logic whenever a signal value changes.

Common use cases include:

- Logging
- API calls
- UI updates
- State synchronization
- Side effects

Effects help create reactive applications while keeping code clean and efficient.

---

## What is an Effect?

An effect is created using the `effect()` function.

### Characteristics

- Defined using `effect()`
- Usually placed inside the constructor
- Accepts a callback function
- Runs automatically when dependent signals change
- Tracks only the signals accessed inside it

### Basic Syntax

```ts
effect(() => {
  // logic here
});
```

---

## Key Rule of Effects

Effects only react to signals that are used inside their callback.

### Example

```ts
effect(() => {
  console.log(this.speed());
});
```

This effect depends only on:

```ts
speed
```

If another signal changes:

```ts
fruit.set('banana');
```

the effect does **not** run.

---

## Initial Execution Behavior

Effects always execute once when they are created.

### Example

```ts
effect(() => {
  console.log(this.speed());
});
```

Output immediately:

```text
0
```

Even if the signal has not changed yet.

---

## Signals Used in the Example

### Speed Signal

```ts
speed = signal(0);
```

Stores vehicle speed.

---

### Fruit Signal

```ts
fruit = signal('apple');
```

Stores fruit name.

---

### Normal Property

```ts
color = 'black';
```

This is **not** a signal.

Used only for UI styling.

## Complete Example

### HTML

```html
<h1>Effects</h1>

<h2 [style.color]="color">
  {{ speed() }}
</h2>

<button (click)="increaseSpeed()">
  Increase Speed
</button>

<hr />

<h2>{{ fruit() }}</h2>

<button (click)="changeFruit()">
  Change Fruit
</button>
```

---

### TypeScript

```ts
import { signal, effect } from '@angular/core';

export class App {

  speed = signal(0);
  fruit = signal('apple');

  color = 'black';

  constructor() {

    effect(() => {

      if (this.speed() > 0 && this.speed() < 80) {
        this.color = 'green';
      }

      if (this.speed() >= 80 && this.speed() < 120) {
        this.color = 'orange';
      }

      if (this.speed() >= 120) {
        this.color = 'red';
      }

      console.log('speed :', this.speed());

    });

    effect(() => {
      console.log(this.fruit());
    });

  }

  increaseSpeed() {
    this.speed.set(this.speed() + 10);
  }

  changeFruit() {
    this.fruit.set('banana');
  }

}
```

## Multiple Effects

Angular allows multiple effects in the same component.

### Example

```ts
effect(() => {
  console.log(this.speed());
});

effect(() => {
  console.log(this.fruit());
});
```

---

### Effect Dependencies

| Effect | Depends On |
|----------|-----------|
| Speed Effect | `speed()` |
| Fruit Effect | `fruit()` |

---

### Result

Changing speed:

```ts
this.speed.set(20);
```

Runs:

```text
Speed Effect ✅
Fruit Effect ❌
```

---

Changing fruit:

```ts
this.fruit.set('banana');
```

Runs:

```text
Fruit Effect ✅
Speed Effect ❌
```

---

## Why This Happens

Angular automatically tracks signal usage.

Example:

```ts
effect(() => {
  console.log(this.speed());
});
```

Angular sees:

```text
This effect uses speed()
```

and subscribes only to that signal.

## Common Uses of Effects

### Logging

```ts
effect(() => {
  console.log(this.user());
});
```

---

### API Calls

```ts
effect(() => {
  const id = this.userId();

  // fetch data
});
```

---

### Navigation

```ts
effect(() => {
  if (!this.isLoggedIn()) {
    this.router.navigate(['/login']);
  }
});
```

---

### Theme Changes

```ts
effect(() => {
  document.body.className = this.theme();
});
```

---

## Important Do's

### ✅ Read Signal Values

```ts
effect(() => {
  console.log(this.count());
});
```

---

### ✅ Perform Side Effects

```ts
effect(() => {
  this.saveToStorage(this.user());
});
```

---

### ✅ Update UI-Related Properties

```ts
effect(() => {
  this.color =
    this.speed() > 100 ? 'red' : 'green';
});
```

---

## Important Don'ts

### ❌ Avoid Updating the Same Signal Inside Its Own Effect

Bad Example:

```ts
effect(() => {
  this.count.set(this.count() + 1);
});
```

Problem:

```text
count changes
↓
effect runs
↓
count changes again
↓
effect runs again
↓
infinite loop
```

---

### ❌ Avoid Recursive Signal Updates

```ts
effect(() => {
  if (this.count() < 100) {
    this.count.set(this.count() + 1);
  }
});
```

This can cause unnecessary re-executions and performance issues.

---

## Effect vs Computed

| Feature | Effect | Computed |
|----------|----------|----------|
| Purpose | Run side effects | Derive values |
| Returns Value | ❌ No | ✅ Yes |
| Reactive | ✅ Yes | ✅ Yes |
| Used for UI Logic | Sometimes | Yes |
| Used for API Calls | ✅ Yes | ❌ No |
| Used for Logging | ✅ Yes | ❌ No |
| Read Only | N/A | ✅ Yes |

---

## Quick Comparison

### Computed

```ts
area = computed(() =>
  this.height() * this.width()
);
```

Creates a derived value.

---

### Effect

```ts
effect(() => {
  console.log(this.area());
});
```

Performs an action when a value changes.

---

## Key Takeaways

- Effects are created using `effect()`.
- They react automatically to signal changes.
- Effects run once immediately when created.
- Only signals used inside an effect trigger it.
- Multiple effects can coexist independently.
- Effects are ideal for logging, API calls, navigation, and side effects.
- Avoid updating the same signal inside its own effect.
- Computed signals derive values, while effects perform actions.
