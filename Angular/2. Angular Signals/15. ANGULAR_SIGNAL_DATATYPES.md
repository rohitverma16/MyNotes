# Signals Data Types in Angular

## Introduction

When working with Angular Signals, there are **two different types of data types** to understand:

1. **Signal Value Data Type** → The type of data stored inside the signal
2. **Signal Data Type** → The type of the signal itself

Both concepts are important and are commonly asked in Angular interviews.

---

## 1. Signal Value Data Type

Every signal stores a value, and that value has a data type.

### Type Inference

By default, TypeScript automatically determines the type from the initial value.

### Example

```ts
count = signal(10);
```

TypeScript infers:

```ts
WritableSignal<number>
```

The signal can only store numbers.

---

## Example: String Signal

```ts
name = signal('Anil');
```

TypeScript infers:

```ts
WritableSignal<string>
```

Only string values can be stored.

---

## Multiple Data Types in a Signal

If a signal needs to hold different types of values, use a union type.

### Example

```ts
data = signal<string | number | boolean | null>(null);
```

### Allowed Values

```ts
this.data.set('Anil');
this.data.set(100);
this.data.set(true);
this.data.set(null);
```

### Not Allowed

```ts
this.data.set({});
```

Because `object` is not included in the defined type.

---

## Why Explicit Types Are Useful

Without explicit typing:

```ts
data = signal('Anil');
```

TypeScript assumes:

```ts
WritableSignal<string>
```

Later this will cause an error:

```ts
this.data.set(100);
```

Because a number cannot be assigned to a string signal.

---

## 2. Signal Data Type

Signals themselves also have types.

Angular provides two main signal types:

### WritableSignal

Can be modified.

### Signal

Read-only signal.

Usually used with computed signals.

---

# WritableSignal

## What is WritableSignal?

A writable signal allows updating its value using:

- `set()`
- `update()`

### Example

```ts
import { signal, WritableSignal } from '@angular/core';

data: WritableSignal<string> =
  signal<string>('anil');
```

---

## Updating Writable Signals

### Using set()

```ts
this.data.set('sidhu');
```

### Using update()

```ts
this.data.update(value => value.toUpperCase());
```

Both are valid because WritableSignal is mutable.

---

# Read-Only Signal

## What is a Read-Only Signal?

Read-only signals cannot be updated directly.

They are generally created using `computed()`.

### Example

```ts
import { Signal, computed } from '@angular/core';

speed: Signal<number> =
  computed(() => 90);
```

---

## Invalid Operations

```ts
this.speed.set(100);
```

❌ Error

```ts
this.speed.update(value => value + 10);
```

❌ Error

Because computed signals are read-only.

---

## Valid Usage

```ts
console.log(this.speed());
```

```html
{{ speed() }}
```

Reading is allowed, writing is not.

---

# set() vs update()

Angular provides two methods to change writable signal values.

---

## set() Method

### Purpose

Used when the new value does **not depend on the previous value**.

### Syntax

```ts
signal.set(newValue);
```

---

### Example

```ts
this.data.set('Sidhu');
```

### What Happens?

```text
Old Value → Replaced
```

Example:

```text
Anil → Sidhu
```

The previous value is completely discarded.

---

## More Examples

```ts
count.set(50);
```

```ts
isLoggedIn.set(true);
```

```ts
theme.set('dark');
```

---

# update() Method

## Purpose

Used when the new value depends on the previous value.

### Syntax

```ts
signal.update(currentValue => {
  return updatedValue;
});
```

---

## Number Example

### Signal

```ts
count = signal(10);
```

### Update

```ts
this.count.update(value => value + 1);
```

### Result

```text
10 → 11
```

---

## Array Example

### Signal

```ts
users = signal(['anil', 'sam']);
```

### Update

```ts
this.users.update(item => [
  ...item,
  'bruce'
]);
```

### Result

```text
['anil', 'sam']
↓
['anil', 'sam', 'bruce']
```

Old data is preserved.

---

## Object Example

### Signal

```ts
user = signal({
  name: 'Anil',
  age: 25
});
```

### Update

```ts
this.user.update(current => ({
  ...current,
  age: 26
}));
```

### Result

```ts
{
  name: 'Anil',
  age: 26
}
```

---

## When to Use set()

Use when:

- Replacing the entire value
- Previous value is not needed

### Examples

```ts
this.name.set('Rahul');
```

```ts
this.count.set(100);
```

```ts
this.theme.set('dark');
```

---

## When to Use update()

Use when:

- New value depends on old value
- Working with counters
- Updating arrays
- Updating objects

### Examples

```ts
this.count.update(v => v + 1);
```

```ts
this.users.update(users => [...users, 'new']);
```

```ts
this.cart.update(items => [...items, product]);
```

---

# Quick Comparison

| Feature | set() | update() |
|----------|--------|----------|
| Uses Previous Value | ❌ No | ✅ Yes |
| Replaces Entire Value | ✅ Yes | ❌ Not necessarily |
| Best For | Simple replacement | Derived updates |
| Arrays | Possible but less common | Recommended |
| Objects | Possible but less common | Recommended |
| Counters | Possible | Recommended |

---

# Signal Types Summary

| Type | Writable | Read Only |
|--------|----------|-----------|
| WritableSignal<T> | ✅ Yes | ❌ No |
| Signal<T> | ❌ No | ✅ Yes |
| Created Using `signal()` | ✅ Yes | ❌ No |
| Created Using `computed()` | ❌ No | ✅ Yes |
| Supports `set()` | ✅ Yes | ❌ No |
| Supports `update()` | ✅ Yes | ❌ No |

---

## Key Takeaways

- Signals have both a **signal type** and a **value type**.
- Value types define what data can be stored.
- Signal types define whether the signal can be modified.
- `WritableSignal<T>` supports `set()` and `update()`.
- `Signal<T>` (computed signals) are read-only.
- Use `set()` when replacing a value.
- Use `update()` when the new value depends on the previous value.
- Explicit typing is useful when a signal can store multiple data types.
