# Introduction to Angular CLI

## What is Angular CLI?

Angular CLI (Command Line Interface) is a tool used to **create, run, build, test, and manage Angular applications** using command-line commands.

It helps developers handle the complete Angular development workflow efficiently.

---

## What Angular CLI is Used For

Angular CLI supports the full application lifecycle:

- Creating Angular applications
- Running applications locally
- Building applications for production
- Testing applications
- Deploying applications

👉 In short, Angular CLI manages everything needed for Angular development.

---

## Why Angular CLI is Important

Angular CLI is used in almost every stage of Angular development.

### Main benefits:

- Quickly create new Angular applications
- Generate components, services, and pipes
- Run development servers
- Build production-ready applications
- Run unit tests

👉 It makes development faster, easier, and standardized.

---

## Installing Angular CLI

Angular CLI is installed using **npm (Node Package Manager)**.

### Important points:

- Node.js is required
- Installed globally using `-g`
- After installation, commands use `ng` instead of `npm`

Example:

```bash
npm install -g @angular/cli
````

---

## Checking Angular CLI Installation

To verify installation, run:

```bash id="k3x9dp"
ng version
```

### This shows:

* Angular version
* Angular CLI version
* Node.js version
* Package manager version
* Operating system details

👉 If this command fails, Angular CLI is not installed correctly.

---

## Angular CLI Help Command

To view all available commands:

```bash id="h7q2mn"
ng help
```

### It shows commands like:

* create application
* build project
* serve application
* generate components/services
* run tests
* update Angular version

---

## Creating a New Angular Application

To create a new project:

```bash id="a9v1lc"
ng new app-name
```

### What it does:

* Creates a new Angular workspace
* Sets up project structure automatically
* Installs required dependencies

---

## Generating Components

Angular CLI is widely used to create components.

### Important points:

* Components are created inside the `app` folder
* `AppComponent` is the root component
* All other components are created inside it

### Commands:

```bash id="c3p8rt"
ng generate component component-name
```

Shortcut:

```bash id="f8k2wd"
ng g c component-name
```

👉 Both commands do the same thing.

---

## Generating Services

Services are used for handling business logic and API calls.

### Key points:

* Services are not automatically placed in folders
* It is recommended to organize them in a separate folder

### Command:

```bash id="s5n2qv"
ng generate service service-name
```

You can also create inside folders:

```bash
ng g s folder/service-name
```

---

## Generating Pipes

Angular CLI also allows creation of pipes.

### Command:

```bash id="p2r8xa"
ng generate pipe pipe-name
```

### Key idea:

* CLI automatically creates required files
* Folder structure can be customized
* Even minor typos still generate files based on command interpretation

---

## Running the Angular Application

To run the app:

```bash id="r8t4lm"
ng serve
```

To automatically open it in the browser:

```bash id="u3k9wp"
ng serve --open
```

👉 This starts the development server and loads the app in the browser.

---

## Building the Application

To create a production build:

```bash id="b6x1qn"
ng build
```

### What it does:

* Compiles the Angular application
* Optimizes performance
* Generates production-ready files in the `dist/` folder
