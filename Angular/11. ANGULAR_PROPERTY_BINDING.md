# Property Binding in Angular

## What is Property Binding?

Property Binding is used to bind **component data** to **HTML element properties**.

### Definition:
- Property Binding connects component data to DOM properties
- It allows dynamic control of HTML elements using TypeScript values

---

## Property Binding Syntax

Property Binding uses **square brackets `[ ]`**:

```html id="pb1"
<button [disabled]="btnDisable">Click Me</button>
````

### Explanation:

* `disabled` → HTML element property
* `btnDisable` → value from TypeScript component

---

## Why Property Binding is Needed?

Some HTML properties (like `disabled`, `readonly`, `src`) require:

* Boolean values
* Dynamic values

👉 Interpolation treats everything as a **string**, not a real boolean

---

## Example 1: Button Disable (Interpolation vs Property Binding)

### ❌ Using Interpolation (Wrong for boolean)

```html id="pb2"
<button disabled="{{btnDisable}}">Click Me</button>
```

### Problem:

* Even if `btnDisable = false`
* Button may still remain disabled

👉 Reason: Interpolation converts value into a string

---

### ✅ Using Property Binding (Correct)

```html id="pb3"
<button [disabled]="btnDisable">Click Me</button>
```

### Behavior:

* `true` → button is disabled
* `false` → button is enabled

---

## Toggle Button Example

### HTML

```html id="pb4"
<button (click)="toggle()">Toggle</button>
```

### TypeScript

```ts id="pb5"
toggle() {
  this.btnDisable = !this.btnDisable;
}
```

### Result:

* Property Binding updates correctly
* Interpolation does not behave properly for boolean logic

---

## Example 2: Input Readonly

### ❌ Using Interpolation

```html id="pb6"
<input type="text" readonly="{{inputReadonly}}">
```

### Problem:

* Even if value is false, input may remain readonly

---

### ✅ Using Property Binding

```html id="pb7"
<input type="text" [readonly]="inputReadonly">
```

### Behavior:

* `true` → input is readonly
* `false` → input is editable

---

## Important Point

Many HTML properties do NOT work correctly with interpolation when dealing with boolean values.

👉 In such cases, Property Binding must be used.

---

## Example 3: Image Source Binding

### ❌ Using Interpolation (not recommended)

```html id="pb8"
<img src="{{url}}" alt="">
```

### ✅ Using Property Binding (recommended)

```html id="pb9"
<img [src]="url" alt="">
```

### Why?

* `src` is a DOM property
* Property Binding ensures correct and safe binding

---

## When to Use Interpolation?

Use Interpolation when:

* Displaying text content
* Showing values in tags like:

  * `<h1>`
  * `<p>`
  * `<span>`
* Writing simple expressions

### Example:

```html id="pb10"
<h1>{{ title }}</h1>
```

---

## When to Use Property Binding?

Use Property Binding when:

* Working with HTML properties
* Handling boolean values
* Binding dynamic values
* Working with:

  * `disabled`
  * `readonly`
  * `src`
  * `value`
  * DOM properties

---

## Key Difference Summary

| Feature         | Interpolation   | Property Binding |
| --------------- | --------------- | ---------------- |
| Syntax          | `{{ }}`         | `[ ]`            |
| Use case        | Display text    | Bind properties  |
| Boolean support | Not reliable    | Fully supported  |
| DOM properties  | Not recommended | Recommended      |

---

## Final Concept

👉 Interpolation = display data
👉 Property Binding = control HTML properties dynamically
