# Styling in Angular

## Introduction

Styling is used to design and improve the appearance of an application.

It controls:

- Colors
- Height and Width
- Layout and Positioning
- Fonts and Typography
- Margins and Padding
- Borders and Shadows

Angular uses **CSS (Cascading Style Sheets)** to style HTML elements and components.

---

# Types of Styling in Angular

Angular supports multiple styling approaches:

1. Component Styles
2. Global Styles
3. Inline Styles
4. Internal Styles
5. External Styles
6. Multiple CSS Files per Component

Each approach has different use cases.

---

# Component Styles

## What Are Component Styles?

Component styles are CSS rules written inside a component's dedicated CSS file.

Examples:

```text
app.component.css
user-data.component.css
profile.component.css
```

These styles apply only to the component they belong to.

---

## Example

### HTML

```html
<h1 class="heading">App Heading</h1>
```

### CSS

```css
.heading {
  background-color: orange;
  color: white;
  padding: 10px;
  border-radius: 10px;
}
```

---

## Output

```text
App Heading
```

with:

- Orange background
- White text
- Rounded corners

---

## Key Points

✅ Applies only to the current component

✅ Prevents style conflicts

✅ Easier maintenance

✅ Angular uses style encapsulation by default

---

## Example Structure

```text
app/
├── app.component.ts
├── app.component.html
└── app.component.css
```

---

# Global Styles

## What Are Global Styles?

Global styles are written in:

```text
src/styles.css
```

These styles affect the entire Angular application.

---

## Example

### styles.css

```css
.heading {
  background-color: orange;
}
```

---

### Any Component

```html
<h1 class="heading">Hello</h1>
```

The style applies everywhere.

---

## When to Use Global Styles

Use for:

- Typography
- Theme colors
- Buttons
- Layout utilities
- Common reusable styles

---

## Benefits

✅ Reusable

✅ Consistent design

✅ Easy theme management

---

## Drawback

❌ Overusing global styles can break component isolation.

---

# Styling Multiple Components

## Common Problem

Same CSS repeated in multiple files:

```text
app.component.css
user-data.component.css
admin-data.component.css
```

Example:

```css
.heading {
  color: white;
  background: orange;
}
```

Repeated everywhere.

---

## Issues

- Code duplication
- Difficult maintenance
- Harder updates

---

## Better Solution

Move shared styles to:

```text
styles.css
```

---

## Benefits

✅ Reusable

✅ Cleaner code

✅ Easier maintenance

✅ Better scalability

---

# Creating Multiple Global CSS Files

Large projects often split styles into multiple files.

---

## Example Structure

```text
src/
├── styles.css
├── buttons.css
├── themes.css
└── common.css
```

---

## Why?

### buttons.css

```css
.btn {
  padding: 10px;
}
```

---

### themes.css

```css
body.dark {
  background: #222;
}
```

---

### common.css

```css
.text-center {
  text-align: center;
}
```

---

# Registering Multiple Global CSS Files

Angular must know about additional CSS files.

---

## angular.json

```json
"styles": [
  "src/styles.css",
  "src/buttons.css",
  "src/themes.css",
  "src/common.css"
]
```

---

## Important

After modifying:

```text
angular.json
```

restart the Angular development server.

```bash
ng serve
```

# Inline Styles

## What Are Inline Styles?

Styles written directly inside HTML elements.

---

## Example

```html
<div style="border:1px solid green;">
  Mid Area
</div>
```

---

## Use Cases

- Quick testing
- Debugging
- Small one-time styles

---

## Drawbacks

❌ Hard to maintain

❌ Not reusable

❌ Clutters HTML

❌ Not recommended for large projects

---

# Internal Styles

## What Are Internal Styles?

Styles written inside a `<style>` tag.

---

## Example

```html
<style>
  .box {
    color: red;
  }
</style>

<div class="box">
  Hello Angular
</div>
```

---

## Why Avoid It?

Angular already provides:

```text
component.css
```

which is cleaner and more maintainable.

---

## Recommendation

Use component CSS files instead.

---

# Inline Template and Styles

## When to Use?

For very small components.

---

## Example

```ts
@Component({
  selector: 'app-user',

  template: `
    <h1>User Data</h1>
  `,

  styles: [
    `
      h1 {
        color: red;
      }
    `
  ]
})
export class UserComponent {}
```

---

## Benefits

✅ Fast setup

✅ Suitable for tiny components

---

## Drawbacks

❌ Hard to read in large components

❌ Not scalable

---

# Loading Multiple CSS Files in One Component

## Default

```ts
@Component({
  styleUrl: './app.component.css'
})
```

---

## Multiple CSS Files

```ts
@Component({
  styleUrls: [
    './app.component.css',
    './common.css'
  ]
})
```

---

## Benefits

- Better organization
- Separation of concerns
- Easier maintenance

---

## Example Structure

```text
profile/
├── profile.component.ts
├── profile.component.html
├── profile.component.css
└── common.css
```

# Styling Best Practices

## ✅ Use Component Styles

Use for:

- Component-specific UI
- Local styling

Example:

```css
.profile-card {
  border: 1px solid gray;
}
```

---

## ✅ Use Global Styles

Use for:

- Buttons
- Typography
- Themes
- Utility classes

Example:

```css
.btn-primary {
  background: blue;
}
```

---

## ✅ Avoid CSS Duplication

Bad:

```css
.heading {
  color: orange;
}
```

Repeated everywhere.

Good:

```css
/* styles.css */
.heading {
  color: orange;
}
```

---

## ✅ Organize Large CSS Files

Split into:

```text
buttons.css
forms.css
themes.css
layout.css
```

---

## ✅ Follow Naming Conventions

Examples:

```css
.btn-primary
.btn-success
.btn-warning
.card-header
.profile-image
```

Consistent naming improves readability.

---

## ❌ Avoid Excessive Inline Styles

Bad:

```html
<div
  style="
    color:red;
    padding:20px;
    border:1px solid black;
  "
>
</div>
```

Better:

```html
<div class="card"></div>
```

```css
.card {
  color: red;
  padding: 20px;
  border: 1px solid black;
}
```

---

# Styling Approaches Comparison

| Styling Type | Scope | Recommended |
|-------------|--------|-------------|
| Component CSS | Single Component | ✅ Yes |
| Global CSS | Entire App | ✅ Yes |
| Multiple Global Files | Entire App | ✅ Yes |
| Inline Styles | Single Element | ⚠ Limited |
| Internal Styles | Component HTML | ❌ Rarely |
| Inline Template Styles | Small Components | ⚠ Small Components Only |

# Interview Questions

### Q1. What is the difference between Component Styles and Global Styles?

**Answer:**

| Component Styles | Global Styles |
|------------------|--------------|
| Apply to one component | Apply to entire application |
| Encapsulated | Shared globally |
| Safer | Can affect all components |

---

### Q2. Where are global styles written?

```text
src/styles.css
```

---

### Q3. How do you add multiple global CSS files?

Register them inside:

```json
angular.json
```

```json
"styles": [
  "src/styles.css",
  "src/buttons.css"
]
```

---

### Q4. Which styling approach is recommended for Angular components?

**Answer:**

Component-specific styles should be placed inside the component CSS file.

---

## Key Takeaways

- Angular supports multiple styling approaches.
- Component styles are isolated and recommended for component UI.
- Global styles are useful for reusable design systems.
- Avoid repeating CSS across components.
- Large projects should organize CSS into multiple files.
- Inline styles should be used sparingly.
- Angular's style encapsulation helps prevent style conflicts.
- Following a structured styling approach improves scalability and maintainability.
