# 🧠 First Problem: Why do we need Promises / Async?

JavaScript normally runs **line by line (synchronously)**.

Example:

```js id="s1"
console.log("1");
console.log("2");
console.log("3");
```

Output:

```
1
2
3
```

---

## ❗ But real-world apps are not like this

Sometimes tasks take time:

* API request (fetch data)
* database call
* loading file
* timer

Example:

```js id="s2"
console.log("Start");

setTimeout(() => {
  console.log("Data loaded");
}, 2000);

console.log("End");
```

Output:

```
Start
End
Data loaded
```

👉 Because JavaScript does NOT wait.

---

# 🧠 So problem is:

We need a way to say:

> “Wait for this task to finish before moving forward”

That is:

# 👉 PROMISES

---

# 📦 1. What is a Promise?

A **Promise is an object that represents a future result**.

Meaning:

> “I don’t have the result now, but I will give it later.”

---

## 🧠 Real-life analogy

You order food:

* Restaurant says: “Your food will be ready in 10 minutes”

That promise has 3 states:

| State     | Meaning       |
| --------- | ------------- |
| Pending   | still cooking |
| Fulfilled | food ready    |
| Rejected  | order failed  |

---

# 🧪 2. Creating a Promise

```js id="p1"
let promise = new Promise((resolve, reject) => {
  resolve("Success!");
});
```

---

## Breakdown:

* `resolve()` → success
* `reject()` → failure

---

## Example with failure:

```js id="p2"
let promise = new Promise((resolve, reject) => {
  reject("Error happened!");
});
```

---

# 🔄 3. Using a Promise (.then / .catch)

## Success case:

```js id="p3"
promise.then(result => {
  console.log(result);
});
```

👉 Output:

```
Success!
```

---

## Error case:

```js id="p4"
promise.catch(error => {
  console.log(error);
});
```

👉 Output:

```
Error happened!
```

---

# 🧠 Real meaning:

| Method | Meaning    |
| ------ | ---------- |
| then   | if success |
| catch  | if error   |

---

# 🌐 4. Real-world Example (VERY IMPORTANT)

Imagine API call:

```js id="p5"
let data = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve("User data loaded");
  }, 2000);
});
```

---

## Use it:

```js id="p6"
data.then(result => {
  console.log(result);
});
```

Output after 2 seconds:

```
User data loaded
```

---

# ⚠️ Problem with Promises

If multiple steps:

```js id="p7"
step1.then(() => {
  step2.then(() => {
    step3.then(() => {
      // messy code
    });
  });
});
```

👉 This becomes messy → called **callback hell**

---

# 🚀 Solution: Async / Await

Async/await makes async code look like normal code.

---

# ⚡ 5. What is Async?

```js id="a1"
async function test() {
  return "Hello";
}
```

👉 async function ALWAYS returns a Promise

---

## Equivalent:

```js id="a2"
function test() {
  return Promise.resolve("Hello");
}
```

---

# ⏳ 6. What is Await?

👉 `await` means:

> “Wait until promise is finished”

---

## Example WITHOUT await:

```js id="a3"
let data = fetch("api");
console.log(data);
```

👉 prints promise (not actual data)

---

## Example WITH await:

```js id="a4"
async function getData() {
  let data = await fetch("api");
  console.log(data);
}
```

👉 Now JavaScript waits

---

# 🧠 Real-life analogy

### Without await:

You start cooking and immediately leave kitchen 😵

### With await:

You wait until food is cooked 🍽️

---

# 🌐 7. Full Real API Example

```js id="a5"
async function loadUser() {
  let response = await fetch("https://api.example.com/user");

  let data = await response.json();

  console.log(data);
}
```

---

# 🔥 Step-by-step:

1. call API
2. wait for response
3. convert to JSON
4. use data

---

# ⚠️ IMPORTANT RULE

👉 `await` only works inside `async` function

---

# ❌ Wrong:

```js id="a6"
let data = await fetch(url); // error
```

---

# ✅ Correct:

```js id="a7"
async function test() {
  let data = await fetch(url);
}
```

---

# 🔥 8. Promise vs Async/Await

| Feature     | Promise       | Async/Await |
| ----------- | ------------- | ----------- |
| readability | medium        | high        |
| structure   | chain (.then) | clean       |
| debugging   | harder        | easier      |

---

# 🧠 9. Angular Connection (VERY IMPORTANT)

Angular uses async everywhere:

### HTTP call:

```ts id="ng1"
this.http.get(url).subscribe(data => {
  console.log(data);
});
```

OR (modern Angular):

```ts id="ng2"
async load() {
  let data = await firstValueFrom(this.http.get(url));
}
```
