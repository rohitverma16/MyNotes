# Interpolation in Angular

## What is Interpolation?

Interpolation in Angular is a feature used to display **TypeScript data inside an HTML template**. It acts as a bridge between:

- Component logic (TypeScript)
- View layer (HTML)

It is one of the most commonly used features in Angular applications.

If a project does not use interpolation, it usually means the app is very simple and does not fully utilize Angular features.

---

## Basic Concept of Interpolation

In a typical Angular component, two files are mainly used:

- **app.component.ts** → Contains logic, variables, functions, API data, and calculations  
- **app.component.html** → Displays data in the browser

👉 Data is written in TypeScript but shown in HTML using interpolation.

Interpolation connects these two layers.

---

## Interpolation Syntax

Interpolation uses **double curly braces**:

```text
{{ }}
````

Anything inside `{{ }}` is evaluated by Angular.

Example:

* A variable `name` defined in TypeScript
* Displayed in HTML using `{{ name }}`

If you write `name` without interpolation, Angular treats it as plain text.

---

## How Interpolation Works Internally

Angular evaluates expressions inside `{{ }}` and checks:

* Is it a property from the component?
* Is it a function?
* Is it a valid JavaScript expression?

### Examples:

```text
{{ name }}       → displays variable value
{{ 10 + 20 }}    → displays 30
{{ Apple }}      → error (if not defined in TypeScript)
```

If Angular cannot resolve it, it throws an error.

---

## Using Multiple Properties

You can display multiple values in the template:

* Name
* Email
* Other variables

Each value must be wrapped individually:

```html
{{ name }}
{{ email }}
```

---

## Using Functions in Interpolation

You can also call functions defined in TypeScript.

### Important points:

* Functions must return a value
* Use `this` to access class properties
* Useful for computed or dynamic values

However:

> Using functions inside interpolation is not recommended for performance reasons unless necessary.

---

## Passing Parameters in Interpolation

Functions can accept parameters inside interpolation:

### Key points:

* Parameters must have proper TypeScript types
* Useful for dynamic calculations
* Example: sum of two numbers

---

## Limitations of Interpolation

### What you CAN do:

* Display variables
* Call functions
* Write simple expressions
* Perform small calculations

### What you CANNOT do:

* Create variables
* Assign values
* Write complex logic
* Declare properties inside HTML

👉 All complex logic should be handled in the TypeScript file.

---

## Interpolation as One-Way Data Binding

Interpolation is a form of **one-way data binding**:

* Data flows from **TypeScript → HTML**
* UI updates automatically when data changes
* No manual DOM manipulation required

If data flows both ways, it becomes two-way binding (not interpolation).

---

## Automatic UI Updates

When a value changes due to:

* Button clicks
* API responses
* Logic execution

Angular automatically updates the UI.

No need for:

* `document.getElementById`
* Manual refresh
* Extra DOM handling

---

## Expressions in Interpolation

Interpolation supports:

### Allowed:

* Mathematical expressions
* Simple conditions
* Small calculations

### Not allowed:

* Complex logic
* Assignments
* Heavy computations

👉 Keep heavy logic in TypeScript, not HTML.

---

## Security Advantage (XSS Protection)

Interpolation provides built-in protection against **Cross-Site Scripting (XSS)**:

* Prevents execution of malicious scripts
* Safely renders user data
* Adds default security layer

---

## Inline Template and Interpolation

For small components, Angular allows inline templates:

Instead of `templateUrl`, you can use:

```ts
@Component({
  template: `<h1>{{ title }}</h1>`
})
```

### Key points:

* Works exactly like external HTML templates
* Interpolation behaves the same
* Common interview topic
