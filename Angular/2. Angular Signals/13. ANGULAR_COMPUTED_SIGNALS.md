# Computed Signals in Angular

## What are Computed Signals?

Computed Signals are **derived signals** whose values depend on one or more other signals.

They automatically recalculate whenever their dependent signals change.

### Key Characteristics

- Created using the `computed()` function
- Depend on other signals
- Read-only
- Cannot be updated directly
- Update automatically when dependencies change
- Improve performance by recalculating only when necessary

---

## Why Computed Signals Are Needed

Consider a rectangle:

```text
Area = Height × Width
```

If `height` or `width` changes, the area should also change.

### Problem with Normal Properties

```ts
height = 100;
width = 20;

area = this.height * this.width;
```

If height changes later:

```ts
this.height = 120;
```

`area` remains unchanged unless manually recalculated.

This leads to:

- Extra code
- Possible bugs
- Manual synchronization

---

## Using Computed Signals

Instead of normal properties:

1. Convert values into signals
2. Create a computed signal
3. Let Angular recalculate automatically

### Example

```ts
height = signal(100);
width = signal(20);

area = computed(() => this.height() * this.width());
```

Now whenever:

- `height` changes
- `width` changes

`area` updates automatically.

---

## Complete Example

### HTML

```html
<h1>Computed Signals</h1>

<h1>Area : {{ area() }}</h1>

<button (click)="handleHeight()">
  Update Height
</button>
```

---

### TypeScript

```ts
import { signal, computed, effect } from '@angular/core';

export class App {

  height = signal(100);
  width = signal(20);

  area = computed(() => this.height() * this.width());

  constructor() {

    effect(() => {
      console.log("is updating?");

      // console.log("area is :", this.area());

    });

  }

  handleHeight() {
    this.height.set(this.height() + 10);
  }

}
```

---

## How It Works

### Initial Values

```ts
height = 100
width = 20
```

Computed value:

```text
area = 100 × 20 = 2000
```

---

### After Clicking Button

```ts
height = 110
```

Computed signal automatically recalculates:

```text
area = 110 × 20 = 2200
```

No manual calculation is required.

## Syntax of Computed Signals

### Basic Syntax

```ts
computed(() => {
  return expression;
});
```

Example:

```ts
total = computed(() => this.price() * this.quantity());
```

---

## Important Rule

Computed signals always use a callback function.

```ts
area = computed(() => this.height() * this.width());
```

The callback tracks all signals used inside it.

Angular automatically knows:

```text
area depends on:
✓ height
✓ width
```

---

## Computed Signals Are Read-Only

### ❌ Not Allowed

```ts
this.area.set(5000);
```

```ts
this.area.update(value => value + 1);
```

This causes an error.

---

### ✅ Allowed

Update the source signals:

```ts
this.height.set(200);
```

or

```ts
this.width.set(50);
```

Computed signal updates automatically.

---

## Effect and Computed Signals

### Example

```ts
effect(() => {
  console.log(this.area());
});
```

Whenever:

```text
height changes
OR
width changes
```

Angular recalculates:

```text
area
```

and runs the effect again.

---

## Important Behavior of effect()

An effect only reacts to signals that are accessed inside it.

### Example

```ts
effect(() => {
  console.log("Hello");
});
```

This effect runs only once because no signal is used.

---

### Reactive Example

```ts
effect(() => {
  console.log(this.area());
});
```

Now the effect depends on `area`.

Whenever `area` changes:

```text
effect runs again
```

---

## Performance Benefit

Computed signals are optimized.

Angular:

- Tracks dependencies automatically
- Recalculates only when needed
- Avoids unnecessary work

Example:

```ts
area = computed(() => this.height() * this.width());
```

If another signal changes:

```ts
count.set(count() + 1);
```

Angular does **not** recalculate `area` because it doesn't depend on `count`.

---

## DOM Update Optimization

### Traditional Approach

A property update may trigger checks across the component.

### Signals Approach

Angular updates only the parts of the UI that use the signal.

Example:

```html
<h1>{{ area() }}</h1>
```

Only this section updates when `area` changes.

---

## Writable Signal vs Computed Signal

| Feature | Writable Signal | Computed Signal |
|----------|----------------|----------------|
| Created With | `signal()` | `computed()` |
| Can Store Value | ✅ Yes | ❌ Derived only |
| Can Use `set()` | ✅ Yes | ❌ No |
| Can Use `update()` | ✅ Yes | ❌ No |
| Depends on Other Signals | ❌ Optional | ✅ Required |
| Automatically Recalculates | ❌ No | ✅ Yes |
| Read Only | ❌ No | ✅ Yes |

---

## Example: Full Name Computed Signal

### Writable Signals

```ts
firstName = signal('Anil');
lastName = signal('Sidhu');
```

### Computed Signal

```ts
fullName = computed(
  () => `${this.firstName()} ${this.lastName()}`
);
```

### Output

```text
Anil Sidhu
```

If:

```ts
this.firstName.set('Rahul');
```

Output automatically becomes:

```text
Rahul Sidhu
```

---

## Key Takeaways

- Computed Signals derive values from other signals.
- Created using `computed()`.
- Always use a callback function.
- Automatically recalculate when dependencies change.
- Cannot be updated directly using `set()` or `update()`.
- Improve performance through dependency tracking.
- Only affected DOM elements update.
- Work seamlessly with `effect()` for reactive programming.
