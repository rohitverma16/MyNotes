# Angular Signals

## What are Signals?

Signals are **reactive wrappers around values**.

They were introduced in Angular relatively recently (around Angular 16) to provide a more efficient and reactive way of managing state.

### Key Idea

A Signal:

- Stores a value
- Tracks when the value changes
- Notifies Angular and TypeScript when updates occur

This makes state management more reactive and efficient.

---

## Normal Property vs Signal

### Normal Property

```ts
data = 10;
```

### Characteristics

- Stores a value directly
- UI updates when the value changes
- TypeScript cannot automatically detect the change

---

### Signal

```ts
count = signal(0);
```

### Characteristics

- Created using the `signal()` function
- Wraps a value inside a reactive container
- Sends notifications whenever the value changes
- Changes can be detected inside TypeScript

---

## Displaying a Normal Property

### TypeScript

```ts
data = 10;
```

### HTML

```html
<h2>{{ data }}</h2>
```

Output:

```text
10
```

---

## Displaying a Signal

### TypeScript

```ts
count = signal(0);
```

### HTML

```html
<h2>Signal : {{ count() }}</h2>
```

### Important

Signals are accessed using parentheses:

```ts
count()
```

because signals behave like functions.

---

## Updating a Normal Property

### HTML

```html
<button (click)="updateData()">Update Data</button>
```

### TypeScript

```ts
updateData() {
  this.data++;
}
```

### Result

- Value increases
- UI updates

### Limitation

TypeScript does not automatically know that `data` changed.

---

## Updating a Signal

### HTML

```html
<button (click)="updateCount()">Update Count</button>
```

### TypeScript

```ts
updateCount() {
  this.count.set(this.count() + 1);
}
```

### Result

- Value increases
- UI updates
- Signal sends change notification

---

## Effect in Signals

### What is Effect?

`effect()` is a reactive Angular feature that runs whenever a signal changes.

It is commonly placed inside the constructor.

---

## Effect Example

```ts
constructor() {
  effect(() => {
    console.log("this is count", this.count());

    if (this.count() === 10) {
      this.count.set(0);
    }
  });
}
```

---

## What Happens Here?

Every time `count` changes:

1. Effect runs automatically
2. Latest value is logged
3. When value reaches `10`
4. Count resets to `0`

---

## Normal Property Inside Effect

```ts
// console.log("this is data", this.data);
```

### Result

- Effect does not react
- No notification is received
- Changes cannot be tracked automatically

---

## Signal Inside Effect

```ts
console.log("this is count", this.count());
```

### Result

- Effect runs whenever count changes
- Changes are detected automatically
- Reactive behavior is achieved

---

## Why Signals are Reactive

Signals automatically react to value changes.

Effects can immediately respond and perform actions such as:

- API calls
- Resetting values
- Redirecting users
- Login/logout handling
- State synchronization

---

## Counter Example Using Signals

### Initial State

```ts
count = signal(0);
```

### Increment Function

```ts
updateCount() {
  this.count.set(this.count() + 1);
}
```

### Effect

```ts
effect(() => {
  if (this.count() === 10) {
    this.count.set(0);
  }
});
```

### Result

```text
0 → 1 → 2 → ... → 9 → 10 → 0
```

Everything happens automatically through reactivity.

## Types of Signals

### 1. Writable Signal

Value can be changed directly.

```ts
count = signal(0);
```

Methods:

```ts
count.set(10);
count.update(value => value + 1);
```

---

### 2. Computed Signal

Read-only signal derived from other signals.

```ts
count = signal(5);

doubleCount = computed(() => this.count() * 2);
```

Output:

```text
count = 5
doubleCount = 10
```

Whenever `count` changes, `doubleCount` updates automatically.

---

### 3. Effect Signal

Created using `effect()`.

```ts
effect(() => {
  console.log(this.count());
});
```

Runs only when dependent signals change.

---

## Change Detection Difference

### Normal Property

```ts
data = 10;
```

- Traditional Angular change detection
- Can affect the entire component

---

### Signal

```ts
count = signal(10);
```

- Updates only where signal is used
- More optimized
- Fine-grained reactivity

---

## Signals vs React State

### React State

```js
const [count, setCount] = useState(0);
```

- State update triggers component re-render

---

### Angular Signals

```ts
count = signal(0);
```

- Updates only dependent UI parts
- More granular updates
- No full component re-render

---

## Syntax Difference

### Normal Property

```ts
this.data = 20;
```

---

### Signal

```ts
this.count.set(20);
```

or

```ts
this.count.update(value => value + 1);
```

## Quick Comparison

| Feature | Normal Property | Signal |
|----------|----------------|----------|
| Reactive | ❌ No | ✅ Yes |
| Change Notifications | ❌ No | ✅ Yes |
| Used with Effect | ❌ No | ✅ Yes |
| Access Syntax | `data` | `count()` |
| Update Syntax | `data = 20` | `count.set(20)` |
| Fine-Grained Updates | ❌ No | ✅ Yes |
| Computed Values | ❌ No | ✅ Yes |

---

## Key Takeaway

Signals are Angular's modern reactive state management solution.

- Store reactive values
- Notify when data changes
- Work with `effect()` and `computed()`
- Provide more efficient change detection
- Update only the parts of the UI that depend on them

👉 Use **Signals** for reactive state and **normal properties** for simple non-reactive data.
