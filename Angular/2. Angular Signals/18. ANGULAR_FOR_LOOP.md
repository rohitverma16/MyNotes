# `@for` in Angular

## Introduction

`@for` is Angular's modern control flow statement used for looping (iteration) inside HTML templates.

It replaces the older:

```html
*ngFor
```

syntax and provides a cleaner, more readable approach.

### Important

- Used only inside HTML templates
- Cannot be used inside TypeScript files
- Designed for rendering lists efficiently
- Works especially well with Angular Signals

---

# Where Can Loops Be Used?

Angular applications commonly use loops in two places:

## 1. TypeScript File

Use normal JavaScript loops:

```ts
for()
```

```ts
for...of
```

```ts
map()
```

```ts
filter()
```

```ts
forEach()
```

Example:

```ts
users.forEach(user => {
  console.log(user);
});
```

---

## 2. HTML Template

Use Angular control flow:

```html
@for
```

Example:

```html
@for (user of users(); track user) {
  <li>{{ user }}</li>
}
```

---

## Rule

✅ Use `@for` only in HTML

❌ Never use `@for` inside `.ts` files

---

# Why Use Signals with `@for`?

Angular now recommends using Signals for state management.

### Benefits

- Better performance
- Faster updates
- Fine-grained reactivity
- Efficient DOM rendering

---

## Example Signal Array

### TypeScript

```ts
users = signal([
  "Anil",
  "Sam",
  "Peter",
  "Bruce"
]);
```

---

## Accessing Signal Values

Signals are accessed as functions:

```ts
users()
```

Not:

```ts
users
```

---

# Using `@for` with a Simple Array

### HTML

```html
<ul>
  @for (user of users(); track user) {
    <li>{{ user }}</li>
  }
</ul>
```

---

## Explanation

### Loop Variable

```ts
user
```

Represents the current item.

---

### Data Source

```ts
users()
```

Returns the array stored inside the signal.

---

### Track Expression

```ts
track user
```

Used to uniquely identify each item.

---

## Output

```text
Anil
Sam
Peter
Bruce
```

---

# Why is `track` Important?

Angular uses tracking to efficiently update the DOM.

Without tracking:

```text
Angular may recreate elements unnecessarily
```

With tracking:

```text
Only changed items are updated
```

---

## Best Practice

Use a unique value:

```ts
track user.id
```

or

```ts
track user.email
```

or

```ts
track user
```

(if values are unique)

---

# Why NOT Use `$index` for Tracking?

Bad Example:

```html
@for (user of users(); track $index)
```

---

## Problem

Indexes change when:

- Items are added
- Items are removed
- Items are reordered

Example:

```text
0 → Anil
1 → Sam
2 → Peter
```

Remove Sam:

```text
0 → Anil
1 → Peter
```

Indexes change.

Angular may incorrectly update the DOM.

---

## Recommended

Use stable unique values:

```html
track user.id
```

---

# Contextual Variables in `@for`

Angular automatically provides special variables inside the loop.

---

## Available Variables

| Variable | Description |
|-----------|-------------|
| `$index` | Current index |
| `$count` | Total items |
| `$first` | True for first item |
| `$last` | True for last item |
| `$even` | True for even index |
| `$odd` | True for odd index |

---

# Using `$index`

### Example

```html
@for (user of users(); track user) {

  <span>{{ $index + 1 }}</span>

  {{ user }}

}
```

---

## Output

```text
1 Anil
2 Sam
3 Peter
4 Bruce
```

---

## Why `+1`?

Indexes start from:

```text
0
```

Adding `1` creates user-friendly numbering.

---

# Using `$count`

### Example

```html
<p>Total Users: {{ $count }}</p>
```

---

### Output

```text
Total Users: 4
```

---

# Using `$first`

### Example

```html
@if ($first) {
  <span style="color:red">
    Admin
  </span>
}
```

---

### Output

```text
Admin
```

Displayed only for the first item.

---

# Using `$last`

### Example

```html
@if ($last) {
  <span style="color:orange">
    Guest
  </span>
}
```

---

### Output

```text
Guest
```

Displayed only for the last item.

---

# Example Using Both

```html
@for (user of users(); track user) {

  @if ($first) {
    <span>Admin</span>
  }

  {{ user }}

  @if ($last) {
    <span>Guest</span>
  }

}
```

---

# Styling Even and Odd Rows

## Example

```html
<li
  [style.backgroundColor]="
    $even ? '#ccc' : '#fff'
  "
>
  {{ user }}
</li>
```

---

## Result

| Row | Color |
|------|--------|
| 1 | White |
| 2 | Gray |
| 3 | White |
| 4 | Gray |

---

## Alternative Using `$odd`

```html
<li
  [style.backgroundColor]="
    $odd ? '#eee' : '#fff'
  "
>
  {{ user }}
</li>
```

---

# Showing Total Count at the End

### Example

```html
@if ($count == $index + 1) {

  <b>
    Total Users {{ $count }}
  </b>

}
```

---

## How It Works

When:

```ts
$count == $index + 1
```

Angular knows:

```text
Current item is the last item
```

---

## Output

```text
Anil
Sam
Peter
Bruce

Total Users 4
```

# Complete Example

## TypeScript

```ts
import { signal } from '@angular/core';

export class AppComponent {

  users = signal([
    "Anil",
    "Sam",
    "Peter",
    "Bruce"
  ]);

}
```

---

## HTML

```html
<ul>

  @for (user of users(); track user) {

    <li
      [style.backgroundColor]="
        $even ? '#eee' : '#fff'
      "
    >

      {{ $index + 1 }}.
      {{ user }}

      @if ($first) {
        <span>(Admin)</span>
      }

      @if ($last) {
        <span>(Guest)</span>
      }

    </li>

    @if ($count == $index + 1) {
      <b>Total Users: {{ $count }}</b>
    }

  }

</ul>
```

# `@for` vs `*ngFor`

| Feature | `*ngFor` | `@for` |
|----------|----------|---------|
| Syntax | Older | Modern |
| Readability | Lower | Higher |
| Performance | Good | Better |
| Control Flow API | No | Yes |
| Angular Recommendation | Legacy | Preferred |

```ts
users = signal([...]);
```

### ✅ Always Use `track`

```html
track user.id
```

### ✅ Use Unique IDs

```html
track user.id
```

Preferred over:

```html
track $index
```

### ✅ Use Context Variables for UI Enhancements

```ts

$last
$even
$odd

```

---

### ❌ Avoid Using `$index` for Tracking

Bad:


# Interview Questions

### Q1. What is `@for`?

**Answer:**

`@for` is Angular's modern control flow statement used for looping through collections inside HTML templates.

---

### Q2. Can `@for` be used in TypeScript files?

**Answer:**

No. `@for` is only used inside Angular templates.

---

### Q3. Why is `track` important?

**Answer:**

It helps Angular efficiently update the DOM by uniquely identifying items.

---

### Q4. Which is better?

```html
track user.id
```

or

```html
track $index
```

**Answer:**

```html
track user.id
```

because IDs remain stable while indexes can change.

---

### Q5. Name some contextual variables available in `@for`.

**Answer:**

```text
$index
$count
$first
$last
$even
$odd
```

## Key Takeaways

- `@for` is Angular's modern replacement for `*ngFor`.
- It is used only inside HTML templates.
- Signals work seamlessly with `@for`.
- Always provide a `track` expression.
- Prefer unique IDs over `$index` for tracking.
- Angular provides useful contextual variables such as `$index`, `$count`, `$first`, `$last`, `$even`, and `$odd`.
- `@for` offers better readability and performance than the older `*ngFor` syntax.
