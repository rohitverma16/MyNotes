# What is a Component in Angular?

## What is a Component?

A component is a part of a larger system.

### Real-life example (Car):
A car is made up of many components:

- Tyres
- Engine
- AC vents
- Dashboard
- Doors

👉 Each of these is a component of the car.

### Key idea:
- A big system is made of components
- A component can also have smaller sub-components inside it

Example:
- Engine → cylinders, belts, pipes, nozzles

---

## Components in Angular

In Angular, everything is built using components.

Examples:

- Header → Component
- Footer → Component
- Login page → Component
- Signup page → Component
- Product listing → Component
- Profile page → Component

👉 Angular applications are basically a collection of components working together.

---

## Why Do We Need Components? (Interview Question)

### Question:
Why not write normal pages? Why use components?

### Answer:
Components provide **reusability**, similar to functions in programming.

### Example:
- One seat design used in multiple parts of a car
- One door design used on all sides

### In Angular:
A single component can be reused in multiple places:

- Login page used in multiple routes
- Profile image used in many sections

---

## Real Example (LinkedIn)

A profile image appears in multiple places:

- Header
- Sidebar
- Comments section

👉 Instead of creating it again and again, Angular uses a reusable component.

---

## Types of Components in Angular

### 1. Root Component
- Only one root component exists in an Angular app
- Default: `AppComponent`
- All other components are placed inside it

---

### 2. Feature Components
Used for specific features like:

- Login
- Signup
- Profile
- Settings
- Product listing

---

### 3. Shared Components
Used in multiple places across the application:

- Buttons
- Search box
- Profile image

---

### 4. Standalone Components
- Before Angular 14 → required NgModule
- Angular 21 → standalone by default
- No need to register in NgModule

---

### 5. Dumb Components
- Only UI-based components
- No logic inside

Examples:
- Static UI blocks
- Animations
- Simple display components

---

### 6. Smart Components
- Contain logic and data handling
- Handle API calls and business logic

Examples:
- Search with auto-suggestions
- Data fetching components

---

## Creating a Component

### Command:

```bash id="c8v3lp"
ng generate component profile
````

Shortcut:

```bash id="q2m7xn"
ng g c profile
```

👉 Both commands create a new component folder.

---

## Component Structure

When a component is created, it includes 4 main files:

### 1. HTML file (UI)

* Buttons
* Input fields
* Headings
* Layout elements

---

### 2. TypeScript file (.ts)

* Variables
* Functions
* API calls
* Business logic

---

### 3. CSS file

* Styling
* Colors
* Layout design

---

### 4. Spec file (.spec.ts)

* Used for unit testing
* Contains test cases
* Not used in every project

---

## Using a Component

### Step 1: Use Selector in HTML

Every component has a selector like:

```html id="p3x9kd"
<app-profile></app-profile>
```

You can use it inside another component’s HTML.

---

### Step 2: Import Component in TypeScript

In the parent component:

```ts id="t6v2qa"
import { ProfileComponent } from './profile/profile.component';
```

## Reusing Components

A component can be reused multiple times:

```html id="r7k2qp"
<app-profile></app-profile>
<app-profile></app-profile>
<app-profile></app-profile>
<app-profile></app-profile>
```

👉 Angular will render the same component multiple times without rewriting code.
