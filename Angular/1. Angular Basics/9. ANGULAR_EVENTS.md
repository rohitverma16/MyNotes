# Angular Events (Input, Click, Change, Mouse Events)

## Introduction

In Angular applications, events are used to handle user interactions.

Common examples include:

- Clicking a button
- Submitting a form
- Typing in an input field
- Detecting value changes

This topic explains how Angular handles events, how to call functions, and how to identify which event was triggered.

---

## Events Covered

- Input event
- Change event
- Click event
- Dropdown change event
- Blur event
- Focus event
- Mouse enter event
- Mouse leave event

---

## HTML Code Example

```html id="events1"
<h1>Events</h1>

<input type="text" (input)="handleEvent($event.type)">
<input type="text" (change)="handleEvent($event.type)">

<button (click)="handleEvent($event.type)">Click</button>

<select (change)="handleEvent($event.type)">
    <option value="">car</option>
    <option value="">bike</option>
    <option value="">mobile</option>
</select>

<input type="text" (blur)="handleEvent($event.type)">
<input type="text" (focus)="handleEvent($event.type)">

<div (mouseleave)="handleEvent($event.type)" class="div1"></div>
<div (mouseenter)="handleEvent($event.type)" class="div2"></div>
````

## TypeScript Code

## Input Event

* Triggered on every keystroke
* Provides real-time updates

```text
Event logged: input
```

---

## Change Event

* Triggered when input loses focus after value change
* Does not trigger on every keystroke

```text
Event logged: change
```

---

## Click Event

* Triggered when a button is clicked

```text
Event logged: click
```

---

---

## Blur Event

* Triggered when input field loses focus
* Happens when user clicks outside the input box

```text
Event logged: blur
```

---

## Focus Event

* Triggered when input field gains focus
* Happens when user clicks inside the input box

```text
Event logged: focus
```

---

## Key Concept – $event.type

Angular uses:

```text
$event.type
```

### Purpose:

* Identifies which event was triggered
* Passed to a single function
* Helps reuse same handler for multiple events

---

## Key Idea

Instead of writing separate functions for each event, Angular allows:

* One function (`handleEvent`)
* Multiple events calling it
* Identification using `$event.type`

👉 This improves code reusability and simplicity.
