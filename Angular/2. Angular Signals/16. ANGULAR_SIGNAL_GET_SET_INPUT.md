# Using Signals in Angular

## Why Use Signals in Angular?

Angular now recommends using **Signals** instead of normal properties for state management.

### Benefits of Signals

- Better performance
- Faster DOM updates
- Reactive programming model
- Cleaner state management
- Fine-grained change detection

### Interview Note

Using only normal properties for state management may indicate familiarity with older Angular patterns.

Modern Angular development increasingly favors **Signals**.

---

## Goal of This Example

In this example, we will learn how to:

- Get a value from an input field using Signals
- Display the value on the UI
- Update the value dynamically
- Reset the value using a button
- Understand common mistakes
- Learn interview-important concepts

# Displaying a Signal Value

### HTML

```html
<p>Value: {{ name() }}</p>
```

### Explanation

Signals are accessed like functions.

```ts
name()
```

returns the current value stored in the signal.

---

## Example

### Signal

```ts
name = signal('Anil');
```

### HTML

```html
<p>Value: {{ name() }}</p>
```

### Output

```text
Value: Anil
```

Whenever the signal changes, Angular updates the UI automatically.

# Getting Input Value Using Signals

## Input Field

### HTML

```html
<input
  type="text"
  [value]="name()"
  (input)="setValue($event.target.value)"
>
```

---

## Explanation

### Property Binding

```html
[value]="name()"
```

Binds the input field value to the signal.

Current signal value is displayed inside the input box.

---

### Input Event

```html
(input)="setValue(...)"
```

Runs whenever the user types.

Unlike `(change)`, `(input)` triggers on every keystroke.

---

### Accessing Input Value

```ts
$event.target.value
```

Gets the latest text entered by the user.

---

### Updating the Signal

```ts
setValue(value)
```

Passes the entered value to a TypeScript function.

---

# Setting Signal Value

### TypeScript

```ts
setValue(val: string) {
  this.name.set(val);
}
```

---

## What Happens?

When the user types:

# Signal Declarations

## Single Value Signal

```ts
user = signal("anil");
```

Stores a single string value.

---

## Array Signal

```ts
users = signal([
  'anil',
  'sam',
  'peter'
]);
```

Stores an array reactively.

---

## Why Signals?

Whenever a signal changes:

```ts
user.set('peter');
```

Angular automatically updates the UI.

No manual DOM manipulation is required.

---

# Updating Signals

## Example Method

```ts
updateSignal() {

  this.user.set("peter");

  this.users.update(
    (prevVal) => [...prevVal, 'bruce']
  );

}
```

---

# Using set()

### Example

```ts
this.user.set("peter");
```

### What Happens?

```text
Old Value → Replaced
```

Before:

```text
anil
```

After:

```text
peter
```

---

# Using update()

### Example

```ts
this.users.update(
  (prevVal) => [...prevVal, 'bruce']
);
```

---

## Explanation

### Previous Array

```ts
['anil', 'sam', 'peter']
```

### After Update

```ts
[
  'anil',
  'sam',
  'peter',
  'bruce'
]
```

---

## Why Use update()?

The new value depends on the previous value.

Angular provides the previous state safely:

```ts
(prevVal) => ...
```

---

## Spread Operator

```ts
[...prevVal, 'bruce']
```

### Meaning

```text
Keep existing values
+
Add a new value
```

# Complete Example

## TypeScript

```ts
import { signal } from '@angular/core';

export class AppComponent {

  name = signal('Anil Sidhu');

  user = signal('anil');

  users = signal([
    'anil',
    'sam',
    'peter'
  ]);

  setValue(val: string) {
    this.name.set(val);
  }

  resetValue() {
    this.name.set('Anil Sidhu');
  }

  updateSignal() {

    this.user.set('peter');

    this.users.update(
      (prevVal) => [...prevVal, 'bruce']
    );

  }

}
```

# Common Mistakes

## ❌ Forgetting Parentheses

Wrong:

```html
{{ name }}
```

Correct:

```html
{{ name() }}
```

Signals must be called like functions.

---

## ❌ Direct Assignment

Wrong:

```ts
this.name = "Angular";
```

Correct:

```ts
this.name.set("Angular");
```

Signals should be updated using `set()` or `update()`.

---

## ❌ Using update() Without Returning Value

Wrong:

```ts
this.users.update(prev => {
  prev.push('bruce');
});
```

Correct:

```ts
this.users.update(prev => [
  ...prev,
  'bruce'
]);
```

Always return the new value.

# Interview Questions

### Q1. Why are Signals preferred over normal properties?

**Answer:**

- Better performance
- Reactive updates
- Fine-grained change detection
- Cleaner state management

---

### Q2. How do you read a Signal value?

```ts
name()
```

---

### Q3. How do you update a Signal value?

```ts
name.set("Angular");
```

or

```ts
name.update(value => value + " JS");
```

---

### Q4. What is the difference between `set()` and `update()`?

| set() | update() |
|---------|---------|
| Replaces value | Uses previous value |
| Simple updates | Derived updates |
| No access to old value | Access to old value |

---

## Key Takeaways

- Signals are Angular's recommended state management approach.
- Signal values are accessed using `signalName()`.
- Use `set()` to replace a value.
- Use `update()` when the new value depends on the previous value.
- Signals automatically update the UI.
- Input fields can be synchronized with signals using property binding and events.
- Signals provide cleaner, faster, and more reactive Angular applications.
