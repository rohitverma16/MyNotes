# Importance of File & Folder Structure in Angular

## Why File & Folder Structure Matters

Understanding the file and folder structure is essential because it helps you:

- Know where to write new code
- Understand where configuration changes are made
- Follow the application execution flow

This is important in all modern frameworks like **Angular**, **React**, and **Vue**.

Without understanding project architecture, becoming an expert is difficult.

---

## Types of Files in an Angular Project

Angular project files can generally be categorized into:

### 1. Sensitive Files
- Small changes can have a large impact on the project
- Should be modified carefully

### 2. Less Important Files
- Rarely used directly
- Their presence or absence usually does not affect the core application

---

## Entry Points in Angular

Angular has two main entry points:

- **HTML Entry Point:** `index.html`
- **TypeScript Entry Point:** `main.ts`

These files initialize and load the application.

---

## package.json

One of the most important files in the project.

### Purpose:
- Stores project metadata
- Defines project configuration

### If deleted or renamed:
- The project will not run

### Contains:
- Project name
- Project version
- Scripts (run, build, test commands)
- Installed libraries and dependencies

---

## Dependencies

### dependencies
- Required for the application to run in production

### devDependencies
- Required only during development
- Example: TypeScript

---

## TypeScript Compilation

- Angular uses **TypeScript**
- Browsers understand only **JavaScript**
- TypeScript is converted into JavaScript during build time

---

## TypeScript Configuration Files

### tsconfig.json
- Main TypeScript configuration file
- Applies to the entire project
- Defines:
  - ECMAScript target version
  - Strict mode rules
  - Error and warning settings

---

### tsconfig.app.json
- Configuration for application code
- Controls:
  - Included files
  - Excluded files
  - Output settings

---

### tsconfig.spec.json
- Used for unit testing
- `spec` files are related to testing configuration

---

## Test Files

- Files with `.spec` extension are test files
- Used for **unit testing**
- Help developers verify code functionality

---

## README.md

- Contains project documentation
- Includes:
  - How to run the project
  - Commands for generating components
  - Build instructions
- Can be updated with project-specific notes

---

## package-lock.json

- Automatically generated file
- Should NOT be edited manually
- Stores exact dependency versions
- Ensures consistency across all developers

---

## angular.json

- Main Angular configuration file
- Contains:
  - Project root settings
  - Source folder configuration
  - Asset paths
  - Styles configuration
  - Build settings

👉 Beginners usually do not modify this file.

---

## node_modules Folder

- Contains all installed libraries
- Created from `dependencies` and `devDependencies`
- Includes nested dependencies

### Important:
- Should NOT be pushed to Git
- Can be recreated using:

```bash
npm install
````

---

## public Folder

Used for static/public assets such as:

* Favicon
* Images
* Extra CSS or JavaScript files

---

## src Folder

* Main source code folder
* Contains all developer-written code

---

## app Folder

Contains Angular application components such as:

* App component
* Login component
* Profile component

Components are created using Angular CLI commands.

---

## index.html

* Main HTML file
* First file loaded in the browser
* Acts as the HTML entry point

---

## main.ts

* Main TypeScript entry file
* Bootstraps the Angular application
* Loads the root component

---

## styles.css

* Global stylesheet
* Applies styles across the entire application

---

## App-Level Files

### app.config.ts

* Application-level configuration
* Defines Angular setup settings

---

### app.component.ts

* Root component file
* Connects HTML, CSS, and logic

---

### app.routes.ts

* Defines application routes
* Used for navigation such as:

  * Home
  * Login
  * Profile

---

### app.component.spec.ts

* Unit test file for App component

---

## Git and Editor Files

### .gitignore

* Specifies files ignored by Git
* Example: `node_modules`

---

### .vscode

* Editor-specific settings
* Does not affect application execution

---

### .editorconfig

* Controls code formatting rules
* Helps maintain consistent code style

---

### .angular

* Stores Angular cache and build-related data
* Improves performance
